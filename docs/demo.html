Self-supervised learning algorithms for imaging without ground truth

ICMS Self-Supervised Machine Learning workshop, 24-25th Feb 2025

Author

Andrew Wang & Julián Tachella

Introduction

We compare self-supervised learning algorithms for solving imaging inverse problems for accelerated MRI reconstruction, using the latest DeepInverse library. Install using pip.

Notes:

  • We assume, in the general case, that measurements are generated from multiple random operators that span the whole space as this is required for many of the algorithms (but not all, including EI).
  • We demonstrate feedforward methods rather than generative models. We also provide self-supervised generative models including GANs. See Example: “Imaging inverse problems with adversarial networks”.
  • We assume noiseless data, however some methods can be easily generalised to noisy scenarios using the SURE family of losses. See Example: “Self-supervised denoising with the UNSURE loss.” and [2].

1. Equivariant imaging methods

1a. Equivariant imaging (EI)

  • Summary
  • Code

The EI self-supervised loss [1] assumes that the set of signals is invariant to a group of transformations in order to learn from incomplete measurement data alone:

\(\mathcal{L}_\text{EI}=\lVert T_g \hat{x} - f(A(T_g \hat{x}), A)\rVert_2^2\)

where \(\hat{x}=f(y, A)\) is a reconstructed signal and \(T_g\) is a transformation sampled at random from a group \(g\sim G\). EI was applied to MRI in [2]. Note that this works even when \(A\) is fixed across the dataset as it constructs multiple virtual operators.

See also: Example - “Self-supervised learning with Equivariant Imaging for MRI”; Example - “Image transformations for Equivariant Imaging”; Docs - deepinv.loss.EILoss.

[1] Chen D., Tachella J. & Davies M., “Equivariant Imaging: Learning Beyond the Range Space”
[2] Chen D., Tachella J. & Davies M., “Robust Equivariant Imaging: a fully unsupervised framework for learning to image from noisy and partial measurements”

loss = [
    dinv.loss.MCLoss(),
    dinv.loss.EILoss(transform=dinv.transform.Rotate())
]
# Set epochs > 0 to train the model
trainer = train(loss, epochs=0)
The model has 2070091 trainable parameters
# Load pretrained model for 50 epochs, test on test dataset, and save sample reconstruction
trainer.ckpt_pretrained = "models/pretrained/ei.pth.tar"
trainer.load_model()

results["ei"] = trainer.test(test_dataloader)

physics.update_parameters(**params)
results["ei"]["sample"] = trainer.model(y, physics)

2. Multi-operator methods

2a. Multi-operator imaging (MOI)

  • Summary
  • Code

The MOI self-supervised loss [4] can be used to learn when signals are observed via multiple (possibly incomplete) forward operators \(\{A_g\}_{g=1}^{G}\), i.e. \(y_i = A_{g_i}x_i\) where \(g_i\in \{1,\dots,G\}\). The loss is constructed as follows:

\(\mathcal{L}_\text{MOI}=\lVert\hat{x} - f(A_g\hat{x},A_g)\rVert_2^2\)

where \(\hat{x}=f(y,A_s)\) is a reconstructed signal (observed via operator \(A_s\)) and \(A_g\) is a forward operator sampled at random from a set \(\{A_g\}_{g=1}^{G}\).

See also: Example - “Self-supervised learning from incomplete measurements of multiple operators.”; Docs - deepinv.loss.MOILoss.

[4] Chen D., Tachella J. & Davies M., “Unsupervised Learning From Incomplete Measurements for Inverse Problems”

loss = [

2b. Multi-operator equivariant imaging (MO-EI)

  • Summary
  • Code

One can combine the multi-operator and equivariant imaging losses to leverage an even greater set of virtual operators (using notation from above):

\(\mathcal{L}_\text{MO-EI}=\lVert T_g\hat{x} - f(A_h(T_g \hat{x}), A_h)\rVert_2^2,\quad A_h\in\{A_h\}_{h=1}^{H},g\in G\)

See also: Docs - deepinv.loss.MOEILoss.

where \(H\) is the set of operators and \(G\) is the group.

3. Measurement splititng methods

3a. Self-supervised data undersampling (SSDU)

  • Summary
  • Code

The SSDU measurement splitting loss [5] splits the measurement \(y\in\mathbb{R}^m\) and forward operator \(A\) into two smaller pairs \((y_1\in\mathbb{R}^{m_1},A_1)\) and \((y_1\in\mathbb{R}^{m_1},A_1)\) to compute the self-supervised loss:

\(\mathcal{L}_\text{SSDU}=\frac{m}{m_2}\lVert y_2 - A_2 f(y_1,A_1)\rVert_2^2\)

where \(f\) is the network, \(A_1 = M_1A, A_2 = M_2A\), and \(M_i\) are randomly generated masks (i.e. diagonal matrices) such that \(M_1+M_2=\mathbb{I}_m\). At inference-time, SSDU simply uses the full measurements \(\hat x=f(y,A)\). Special cases include Artifact2Artifact [6] where \(M_1,M_2\) are independent in the case of sequentially arriving data, and Phase2Phase [7].

See also: Example - “Self-supervised learning with measurement splitting”; Example - “Self-supervised MRI reconstruction with Artifact2Artifact”; Docs - deepinv.loss.SplittingLoss; Docs - deepinv.loss.Artifact2ArtifactLoss; Docs - deepinv.loss.Phase2PhaseLoss.

[5] Yaman B. et al., “Self-supervised learning of physics-guided reconstruction neural networks without fully sampled reference data”
[6] Liu J. et al., “RARE: Image Reconstruction using Deep Priors Learned without Ground Truth”
[7] Eldeniz C. et al., “Phase2Phase…”

3b. Noise2Inverse

  • Summary
  • Code

The Noise2Inverse framework [8] extends SSDU [5] during inference by averaging the model over multiple realizations of the random splitting, i.e.

\(\hat{x} = \frac{1}{N}\sum_{i=1}^N f(y_1^{(i)},A_1^{(i)})\)

where \(N\) is the number of Monte-Carlo samples. The training loss function remains the same as SSDU.

[8] Hendriksen A., Pelt D., Batenburg K., “Noise2Inverse: Self-supervised deep convolutional denoising for tomography”

loss = dinv.loss.SplittingLoss(mask_generator=mask_generator, eval_split_input=True, eval_n_samples=20)
# Set epochs > 0 to train the model
